# Supplementary data — `seed_dispersion_30seeds.csv`

## What this is

The seed-dispersion table backing **§4.4 of the consolidation paper**. Reports the Fano factor of the pentachoric Floquet protocol on four cell topologies, across 30 PRNG seeds, at the headline operating point (τ = 5, ε = 0.10, 20 000 shots per seed).

Generated by `simulations/seed_sweep_driver.py` in this repo, which wraps `simulations/sim_scaling_comparison.py` (frozen copy of `willow_hardware_merkabit/simulations/sim_scaling_comparison.py`).

## Why it exists

Paper 26 [15] reported the headline result `F = 0.285` on the 4×4 square cell at τ = 5, ε = 0.10, computed from a single PRNG seed (`seed=42`). A pre-submission methodological audit identified that single-seed Monte Carlo claims are vulnerable to a cherry-picked-seed objection. This CSV provides the seed-averaged ground truth: the seed=42 value was a low-tail draw at −3.25σ from the 30-seed mean. The consolidation paper §4.4 reports the seed-averaged value `F = 0.370 ± 0.026` and explicitly notes the correction.

## Schema

| Column | Type | Description |
|---|---|---|
| `seed` | int | PRNG seed value (1..30) |
| `cell` | str | One of `hex-r1-7n`, `sq-3x3-9n`, `hex-r2-19n`, `sq-4x4-16n` |
| `tau` | int | Number of Floquet rounds (= 5 throughout) |
| `epsilon` | float | Per-node, per-step error injection probability (= 0.10 throughout) |
| `shots` | int | Monte Carlo shots per (seed, cell) configuration (= 20 000 throughout) |
| `fano_paired` | float | Fano factor with paired (P-gate) chirality |
| `fano_control` | float | Fano factor with unpaired (chirality = 0) control |
| `detection_paired` | float | Detection rate (fraction of shots with ≥ 1 ancilla firing), paired |
| `mean_weight_paired` | float | Mean per-shot syndrome weight, paired |

Total rows: 4 cells × 30 seeds = 120.

## Summary statistics (also reported in §4.4)

| Cell | F_paired (mean ± std) | 95% CI | F_control (mean ± std) |
|---|---|---|---|
| `hex-r1-7n` | 0.4475 ± 0.1056 | [0.2873, 0.6421] | 1.2523 ± 0.1686 |
| `sq-3x3-9n` | 0.3581 ± 0.0657 | [0.2680, 0.4897] | 1.3992 ± 0.1972 |
| `hex-r2-19n` | 0.4477 ± 0.0598 | [0.3553, 0.5592] | 1.3901 ± 0.0942 |
| `sq-4x4-16n` | **0.3703 ± 0.0258** | **[0.3320, 0.4226]** | 1.4402 ± 0.1497 |

The 4×4 square cell has both the lowest mean F_paired and the lowest seed-dispersion of any cell — i.e. it is *both* the tightest *and* the most stable.

Single-seed reference (seed = 42): paper 26 values reproduce as `hex-r1-7n: 0.539 (paper 0.522)`, `sq-3x3-9n: 0.348 (paper 0.343)`, `hex-r2-19n: 0.399 (paper 0.397)`, `sq-4x4-16n: 0.286 (paper 0.285)`. Three of four are within ±1σ of the seed-averaged mean; the 4×4 single-seed value is at −3.25σ.

```bash
cd ../simulations

```

# Supplementary data — `angle_perturbation_1000.csv`

## What this is

The angle-table perturbation control. Tests whether the merkabit's predicted observables (P(|00⟩) at n = 13 ≈ 0.6964, π-lock ⟨ZZ⟩ at n = 12 ≈ 0.4474, etc.) are SPECIFIC to the merkabit Table 1a multipliers, or whether any 15-parameter table with similar magnitudes would give the same values.

Generated by `simulations/angle_perturbation_control.py`. Self-contained (numpy only); ~30 seconds wall time.

## Method

Computes the dual-spinor Floquet unitary explicitly (4 × 4 matrices over 12 steps, repeated up to 60 steps) for three perturbation distributions of the 15-parameter Table 1a multipliers:

- **`shuffle`** — randomly permute the 15 merkabit multipliers (preserves the multiset, breaks the structural assignment).
- **`random`** — replace each multiplier with iid Uniform(0.4, 1.8) (matches the original numerical range).
- **`small`** — add Gaussian noise (σ = 0.1) to each multiplier (tests robustness to ~10% drift).

1 000 trials per perturbation type × 3 types = 3 000 rows.

## Schema

| Column | Type | Description |
|---|---|---|
| `trial` | int | Trial index (1..1000) |
| `perturbation` | str | One of `shuffle`, `random`, `small` |
| `P_at_n12` | float | P(|00⟩) at end of one Floquet cycle |
| `P_at_n13` | float | P(|00⟩) at the predicted local-minimum step (paper Table 7 obs 4: ideal 0.6964) |
| `P_at_n27` | float | P(|00⟩) at second local-minimum (paper §3.4: ideal 0.6832) |
| `P_at_n37` | float | P(|00⟩) at first quasi-period peak (paper §3.4: ideal 0.9156) |
| `P_at_n39` | float | P(|00⟩) at second quasi-period peak (paper Table 7 obs 5: ideal 0.9090) |
| `ZZ_at_n12` | float | ⟨Z⊗Z⟩ at end of one cycle (paper §3.2 π-lock: ideal 0.4474) |

## Headline finding

The merkabit angle table is **specifically discriminative for the high-recurrence peaks** but **typical for the low-recurrence observables**.

| Observable | Merkabit ideal | Random-table mean | Random-table std | Merkabit's percentile |
|---|---|---|---|---|
| P at n = 12 (cycle close) | 0.6968 | 0.7394 | 0.142 | 32% |
| P at n = 13 (local min) | 0.6964 | 0.7388 | 0.156 | 33% |
| P at n = 27 (local min) | 0.6832 | 0.6931 | 0.166 | 41% |
| ⟨ZZ⟩ at n = 12 (π-lock) | 0.4474 | 0.5199 | 0.226 | 35% |
| **P at n = 37 (peak)** | **0.9156** | **0.7133** | **0.204** | **86%** |
| **P at n = 39 (peak)** | **0.9090** | **0.6638** | **0.194** | **94%** |

The two quasi-period peaks (n = 37, n = 39) sit in the **top decile** of the random-table distribution. The local-minimum and π-lock observables sit near the median.

## Interpretation

The merkabit angle table is not uniquely required to produce a Floquet protocol with a P(|00⟩) ≈ 0.7 local minimum near n = 13, or a ⟨ZZ⟩ ≈ 0.45 π-lock. Random tables with similar magnitudes give similar values for these observables. **A Willow measurement of P(n = 13) ≈ 0.70 ± 5% is therefore consistent with both the merkabit angle table and many other similar-magnitude tables; it does not uniquely identify the merkabit prediction.**

The merkabit angle table **is** specifically discriminative for the high-recurrence peaks. P(n = 37) and P(n = 39) sit in the top 10–15 % of distributions under random or shuffled multiplier perturbations. Random tables typically give P(n ≥ 30) values in the 0.6–0.8 range; the merkabit gives > 0.9. **A Willow measurement of P(n = 37) > 0.85 is strong evidence specifically for the merkabit angle table.**

The "small" perturbation result (Gaussian noise σ = 0.1 on each multiplier) shows the merkabit observables are **stable to ~10 % multiplier noise** (std of perturbed observables: 0.04–0.10 for low-n, 0.06–0.09 for the high-recurrence peaks). The predictions are not fine-tuned: they survive realistic calibration drift on hardware.

## Implication for the consolidation paper §5

The most discriminating Willow observables, in order of evidential strength for the angle-table-specialness claim:

1. **P(n = 39) ≈ 0.91 ± 0.05** (Table 7 obs 5) — **most discriminating**, merkabit at 94th percentile vs random tables.
2. **P(n = 37) ≈ 0.92** (paper §3.4) — second-most discriminating, merkabit at 86th percentile.
3. **P(n = 13) ≈ 0.70 ± 5%** (Table 7 obs 4) — least discriminating, merkabit at 33rd percentile (similar tables give similar values).
4. ⟨ZZ⟩ π-lock ≈ 0.45 (§3.2) — also non-discriminating.

A Willow run that lands all six pre-registered point predictions within their thresholds is consistent with the merkabit angle table. The two high-recurrence peaks (observables 5 and the §3.4 n = 37 value) carry most of the model-discrimination weight; the low-recurrence ones are robust to angle-table choice.
